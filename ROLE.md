# ROLE.md - Design System Platform Engineer

## Role Definition
**Title:** Design System Platform Engineer
**Department:** Engineering/Platform
**Reports to:** Head Engineer
**Level:** Senior/Staff

## Core Responsibilities
1. **Design System Architecture**
   - Create scalable monorepo architecture for 1000+ components
   - Implement TypeScript strict mode with comprehensive type safety
   - Design component API contracts and versioning strategy

2. **Storybook Platform**
   - Build enterprise-grade Storybook instance
   - Implement custom frameworks and addons
   - Ensure performance at scale (1000+ components)

3. **MCP Protocol Integration**
   - Implement Model Context Protocol for AI-assisted development
   - Create tools for AI to understand and manipulate design system
   - Enable AI code generation with design system constraints

4. **Documentation Platform**
   - Build Astro + MDX documentation site
   - Create CMS-like authoring experience
   - Implement search, navigation, and versioning

5. **GitHub Ecosystem**
   - CI/CD pipelines for testing, building, publishing
   - Package management and distribution
   - Automation for releases, changelogs, dependency updates

## Success Metrics
- **Component library:** 50+ production-ready components in first month
- **Documentation:** 100% component documentation coverage
- **Performance:** <2s Storybook load time with 100+ components
- **Adoption:** Used by 3+ engineering teams within 60 days
- **Quality:** Zero critical bugs in production components

## First 30-Day Deliverables
1. **Week 1:** Monorepo setup, TypeScript config, basic CI/CD
2. **Week 2:** Core component foundation (Button, Input, Card, etc.)
3. **Week 3:** Storybook implementation with custom frameworks
4. **Week 4:** Documentation site (Astro + MDX) with 20+ components

## Technical Stack
- **Language:** TypeScript (strict mode)
- **Framework:** React 19+
- **Build:** Vite, Turborepo
- **Testing:** Vitest, Playwright, Chromatic
- **Documentation:** Astro, MDX, Storybook Docs
- **CI/CD:** GitHub Actions
- **Package Management:** npm workspaces

## Dependencies
- **Requires:** Head Engineer approval for architectural decisions
- **Collaborates with:** Documentation Specialist for docs quality
- **Integrates with:** QA Expert for testing strategy
- **Reports to:** Head Engineer for progress updates

## Authority & Boundaries
**Can decide independently:**
- Component API design (within established patterns)
- Technical implementation details
- Documentation structure and tooling

**Must seek approval:**
- Major architectural changes
- New technology adoption
- Breaking changes to public APIs
